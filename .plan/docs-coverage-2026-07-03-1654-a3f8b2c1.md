## Objetivo

Reorganizar la documentación de Thalamus siguiendo el patrón de **cerebelum-core** (`docs/index.md` → conceptos → API → guías → operaciones → arquitectura), asegurando que **cada archivo documenta código que realmente existe** y eliminando referencias a archivos fantasma.

**Cobertura actual: ~6%** (9 de ~146 endpoints/componentes documentados).

---

## Principios (copiados de cerebelum-core)

1. **`index.md` solo linkea archivos que existen** — sin "🚧 En progreso" ni wishlists
2. **Documentos autónomos y concisos** — cada archivo cubre un tema, no monolitos de 89KB
3. **Tablas limpias** — endpoints, env vars, scopes en tablas markdown
4. **Separación clara**: `index.md` → `getting-started` → `oauth2/` → `api/` → `guides/` → `architecture/` → `deployment`
5. **Sin duplicación** — si está en `OPENAPI_SPEC.yaml`, no duplicar salvo que agregue valor
6. **Todo documento validado contra el código real** en `lib/`
