## Cambios respecto a v1

1. **5 personas en vez de documentación monolítica**: Dev integrando, Agente de código, DevOps on-prem, Admin, Arquitecto
2. **Nueva sección `agents/`**: CLI + skills para agentes de código que usan Thalamus cloud
3. **`index.md` y `getting-started.md` son routers por persona** (patrón cerebelum-core "Two Modes", extendido a 5)
4. **Referencia OAuth2 y API es compartida** — sirve para cloud y on-prem

---

## Las 5 personas

| # | Persona | Necesidad | URL base | Sección clave |
|---|---|---|---|---|
| 🟦 | **Dev integrando** | Mi app necesita login OAuth2 | `auth.zea.cl` | `oauth2/`, `tutorials/` |
| 🤖 | **Agente de código** | Necesito tokens para actuar como agente, skills/scopes | `auth.zea.cl` | `agents/`, `oauth2/agent-tokens.md` |
| 🟢 | **DevOps on-prem** | Quiero correr Thalamus en mi infra | `localhost:4000` | `deployment.md`, `configuration.md`, `architecture/` |
| 🟣 | **Admin** | Administro usuarios, orgs, clientes, roles | Dashboard o API | `api/`, `guides/` |
| 🟡 | **Arquitecto/Tech Lead** | Entender arquitectura, decisiones de diseño | — | `architecture/`, `oauth2/overview.md` |

## Prioridades (v2)

### 🔴 P1 — OAuth2 Core ✅ COMPLETADO (8/8)
Sin esto, thalamus es indocumentable como servicio OAuth2.

### 🟠 P1.5 — Agents (3 archivos NUEVOS)
Para agentes de código que usan Thalamus cloud. Sin esto, el caso de uso "agente" está ciego.

### 🟡 P2 — API REST (11 archivos)
CRUD de usuarios, organizaciones, roles, MFA, etc. El 80% de las operaciones diarias.

### 🟢 P3 — Arquitectura + Configuración + Guías (5 archivos)
Clean Architecture, deployment, SAML, dashboard.

### ⚪ P4 — Limpieza y cierre (6 archivos)
`index.md`, `getting-started.md`, archivar `INTEGRATION_GUIDE.md`, eliminar `README.md` viejo.

## Reglas de ejecución (sin cambios)

1. Cada archivo se valida contra el código real — leer el controller/ex antes de escribir
2. Formato cerebelum-core: tabla de endpoints → parámetros → ejemplos curl → error codes
3. Nada de "🚧 En progreso" — si no existe en código, no se documenta
4. Commits atómicos
5. `index.md` y `getting-started.md` se crean al final
